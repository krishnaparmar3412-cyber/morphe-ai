## ⚙️ Understanding the Workspace

When you open the application, you see a clean dashboard. The workspace splits into three main areas:

1. The Import Area: This is where you drag and drop your Android APK files. The software immediately begins a scan to identify the file structure.

2. The AI Pipeline Console: This area displays the progress of the analysis. It shows what the AI finds, such as specific functions or text strings within the code.

3. The Patching Window: This is where you decide which modifications to apply. You can select pre-built patches or request the AI to generate a specific fix.

## 🧠 How the AI Analysis Works

Modern Android apps contain thousands of lines of hidden code. Morphe-ai breaks this code into readable parts. Instead of reading through massive files, you see a report of what the app does. 

If you want to remove an advertisement or change a setting within an app, you select the target area. The internal agent scans the file and finds the correct lines of code to rewrite. It handles the complex logic required to keep the app working after the change.

## 🚀 Running Your First Patch

After importing your APK file, follow this process to make a modification:

1. Let the scan finish. You will see a list of functions in the side menu.

2. Click on the function you want to modify.

3. Use the search box to find specific keywords if the list is long.

4. Select the desired patch from the suggestion library.

5. Press the Apply button located at the bottom of the screen.

6. Wait for the processor to rebuild the file.

7. The software will notify you when the new, modified APK file is ready.

## 📝 Frequently Asked Questions

Can I use this on a Mac?
This specific version works only on Windows.

Does it delete my original file?
No, the software creates a new, modified version of your APK and leaves the source file untouched.

Do I need to know how to code?
No. The interface handles the technical code changes automatically.

Where are my files saved?
The program saves all output files in your Documents folder under a subfolder named MorpheData.

What if the app fails to open after patching?
Usually, this happens if the patch conflicts with security settings inside the app. You can revert to the original version by clicking the Restore button in your project history.
